Getting started

The @angular-extensions/model can be used in both projects using Angular CLI and plain Angular projects with custom build


  • Use with Angular CLI
  • Use in plain Angular proect


1. Install @angular-extensions/model

Enter Angular CLI project and run ng add @angular-extensions/model. The library uses new providedIn: 'core' tree-shakeable providers so we can start injecting and using ModelFactory out of the box without need to import any modules or register any providers.


2. Generate new model service

Create new model service using Angular CLI and provided schematics by running ng g @angular-extensions/model:model examples/todo --items. This will generate service and corresponding test files with basic implementation. The --items flag causes that the model service will work with the collection of items which means it will expose Observable <Todo[]> as its data.


3. Generate new component and use the model service

Create new component using Angular CLI and built in Angular schematics by running ng g component examples/todo --inline-template.

Adjust component implementation as in the code example below and then use our new <app-todo></app-todo> component in the template of the existing app.component.html (or wherever it suits you).

Please mind that you might be using different application prefix than app- so adjust accordingly.

4. Implement additional model service methods

Create additional model service methods following pattern used in the one existing method.

Every model method consists of three main parts:

  1. Get current model state with const state = this.model.get()
  2. Mutate current model state based on your needs
  3. Set the new model value using this.model.set(state)

Please, have a look into implementation of Live Demo to see more examples of model service methods.
