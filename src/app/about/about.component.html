Angular Model Pattern


Simple state management with minimalistic API, one way data flow, multiple models support and immutable data exposed as RxJS Observable

Install using npm npm i -S ngx-model
or yarn yarn add ngx-model (less than 1kB)

Features

photo_size_select_small Minimalistic API

All you need are two methods and one property! Subscribe to state exposed as RxJS Observable .data$ property, .get() current state in your service methods, update it and .set(newState) to notify all subscribed components.

autorenew One way data flow

One way data flow has proved to be the most effective and understandable way of managing application state. Components simply subscribe to model state exposed as RxJS Observable with | async pipe in their template or explicitly by calling .subscribe() in ngOnInit().

content_copy Immutable model

Stop worrying about unintended side-effects! Your data is accessible through an RxJS Observable which clones underlying data by default. This prevents accidental data mutations by model consumers.

Immutability can be overridden by using .createMutable(data) or .createWithCustomClone(data, cloneFn) factory methods.

share Sharing state by multiple component

Share state between multiple components by injecting one service and subscribing to the same model. This makes state sharing independent from component's DOM location (no direct parent child eventing).

folder One model per feature

Create multiple models to separate your state into manageable chunks. Combine state from multiple models using RxJS .combineLatest() or one of the many available operators.

bug_report Easy debugging

Model is updated by explicit calls to service methods which are easy to understand and debug. Step through the call stack from component event handler to model data setter - one function call at the time!

favorite Angular way (AoT compatible)

Create multiple instances of your model by using standard Angular dependency injection mechanism and build in factory support.
